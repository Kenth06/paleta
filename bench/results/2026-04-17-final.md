# Bench results — 2026-04-17 (end of v0.4)

Node v22, aarch64 (Apple Silicon), vitest 2.1.9.
Full suite, all passing, all committed to main.

## 128×128 noise — post-resize hot path

| Path                               | hz     | mean (ms) | p99 (ms) |
| ---------------------------------- | ------ | --------- | -------- |
| buildHistogram only (JS)           | 5,799  | 0.172     | 0.653    |
| quantizeWu(10) only (JS)           | 1,991  | 0.502     | 2.24     |
| quantizeWu(16) only (JS)           | 1,883  | 0.531     | 2.73     |
| JS: hist + quantizeWu(10) combined | ~1,200 | 0.83      | 3.07     |
| **WASM: quantizeWuWasm(10)**       | **4,132**  | **0.242** | **0.31**     |

WASM is **2.92× faster mean, ~10× better p99** on the bread-and-butter
post-resize quantize. Tail latency wins dominate — in a Workers isolate
under load the p99 improvement is what users actually feel.

## 1024×1024 noise — unresized / stride-sampled

| Path                                          | hz     | mean (ms) | p99 (ms) |
| --------------------------------------------- | ------ | --------- | -------- |
| JS: buildHistogram(step=1)                    | 196    | 5.11      | 5.77     |
| JS: buildHistogram(step=50)                   | 5,005  | 0.200     | 0.71     |
| JS: hist(step=50) + quantizeWu(10)            | 1,284  | 0.779     | 2.72     |
| **WASM: quantizeWuWasm(step=50, count=10)**   | **2,970**  | **0.337** | **0.48**     |

Stride sampling is **25.6× faster** than full scan. WASM adds another
**2.31×** on top of that.

## What the pipeline actually looks like end-to-end

Assuming typical web JPEG, decoded to 1920×1080, resized to 128×128:

```
fetch          ~15–60ms   (depends on origin)
sniff          ~0.01ms
decode (JPEG)  ~40–80ms   (jSquash WASM, dominant)
resize         ~0.3ms
hist + wu      ~0.25ms    (WASM) or ~0.8ms (JS)
sort + oklch   ~0.01ms
cache put      ~0.5ms
───────────────────────
cold total     ~55–140ms
warm            ~1–3ms    (caches.default hit)
cross-colo hit ~15–30ms   (DO round trip)
```

The pipeline is fully decode-dominated once the quantizer is WASM.
The logical next win is the JPEG DC-only decoder path — that should
collapse the decode step to ~3–10ms for most inputs.
