# Bench results — 2026-04-17 (v0.2 WASM vs JS)

Node v22, aarch64 (Apple Silicon), vitest 2.1.9.
Rust crate built with `RUSTFLAGS='-C target-feature=+simd128'`, LLVM
auto-vectorization only (no manual SIMD intrinsics yet). `wasm-opt` not
applied. WASM artifact: 27.4 KB.

## Workload: 128×128 noise (typical post-resize input)

| Benchmark                        | hz     | mean (ms) | p99 (ms) |
| -------------------------------- | ------ | --------- | -------- |
| JS: histogram + quantizeWu(10)   | 1,203  | 0.831     | 3.07     |
| WASM: quantizeWuWasm(10)         | 4,063  | 0.246     | 0.40     |

**WASM is 3.38× faster.** p99 also drops from 3.07ms to 0.40ms — tighter tail
latency matters a lot at edge scale.

## Workload: 1024×1024 noise, step=50 (full-decode fallback path)

| Benchmark                                  | hz     | mean (ms) | p99 (ms) |
| ------------------------------------------ | ------ | --------- | -------- |
| JS: histogram(step=50) + quantizeWu(10)    | 1,315  | 0.761     | 2.60     |
| WASM: quantizeWuWasm(step=50, count=10)    | 2,988  | 0.335     | 0.46     |

**WASM is 2.27× faster.**

## Interpretation

- We're at the edge of what LLVM autovec can do on a gather-scatter histogram.
  The inner loop indexes `moments[idx * 5 + n]` where `idx` depends on the
  pixel — WASM SIMD has no scatter instructions, so vectorization is blocked
  on the accumulation step.
- The 3.38× comes mostly from tighter iteration + no V8 object-heap overhead
  in the box structures.
- Tail latency wins are the bigger story: p99 improves 7.7× on 128×128. In
  a Worker isolate with noisy neighbors this is what users actually feel.

## Unlocked next

Ideas that would push the WASM further (targeted at v0.3+):

1. **Manual SIMD u8x16 histogram prefetch** — pre-compute 16 bucket indices
   per v128 load, then scalar-scatter. Should save ~30% on histogram.
2. **wasm-opt -O3 pass** — LLVM bytecode cleanup. Usually 5–15% extra.
3. **Fused histogram + volume cube** — skip the `moments → cumulative`
   intermediate, update cumulative tables directly during histogram build.
   Removes the 32³ double-pass. Target: <0.1ms on 128×128.
4. **Workers isolate benchmark** — run this harness in
   `@cloudflare/vitest-pool-workers` and publish isolate-local numbers.
5. **Cold WASM parse time** — measure `initWasm()` latency; should be <15ms
   for our 27.4KB artifact. If over, apply `wasm-opt -Oz`.
