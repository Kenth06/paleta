# Bench results — 2026-04-24 cache-hit (workerd isolate)

Completes the perf-target table by measuring the `caches.default` hit
path end-to-end inside a real `workerd` process (wrangler 4.83.0 dev,
`--local`). Uses the same inlined 640×480 4:2:0 JPEG fixture as the
2026-04-20 isolate run, same `/bench` endpoint in `examples/minimal-
worker`, now extended with `path=cache`.

## How it works

`path=cache` primes `caches.default` with one miss, verifies the next
call returns `meta.path === "cache-hit"` (so we're not silently
reporting the uncached number), then measures N hit iterations:

```
for i in 0..N:
  t0 = performance.now()
  getPalette(fixture, { cache: caches.default, cacheKey, ... })
  runs.push(performance.now() - t0)
```

The fixture has no source URL (it's raw bytes), so an explicit
`cacheKey` is required — the pipeline's default cache-key builder
falls back to skipping the cache when `norm.sourceUrl` is empty.

## Numbers

| Path     | iters | mean (ms) | p95 | p99 | max | hz (sustained) |
| -------- | ----- | --------- | --- | --- | --- | -------------- |
| cache    | 5000  | **0.164** | 1   | 1   | 9   | **6,098**      |
| cache    | 2000  | 0.160     | 1   | 1   | 4   | 6,270          |
| DC-only  | 500   | 0.680     | 1   | 2   | 3   | 1,471          |
| full     | 500   | 2.642     | 3   | 9   | 13  | 379            |

- Cache hit is **~4.1× over DC-only** and **~16× over full-decode**
  in the same isolate.
- Sustained **~6,100 cache-hit req/sec per isolate** at these
  CPU budgets.

## Caveat: timer resolution

`workerd` rounds `performance.now()` to 1 ms without the paid
`high_precision_performance_now` compat flag, which is why p50/p95/p99
show as integer milliseconds. The `mean` is an arithmetic average over
thousands of samples, so it resolves sub-ms reliably. Single-iteration
p99 should be treated as an upper bound, not a precise figure.

The cache-hit single-iteration cost is clearly <1 ms (otherwise the
mean would not be 0.16), but any claim tighter than "mean 0.16 ms"
exceeds what the timer can measure in this configuration.

## How it lines up with the perf targets

CLAUDE.md's refreshed table sets cache-hit at 1–3 ms. Measured:

| Path                  | Target cache hit | Measured (workerd) | Ratio        |
| --------------------- | ---------------- | ------------------ | ------------ |
| EXIF-thumb / DC / full| 1–3 ms           | **0.164 ms**       | ~6–18× under |

The 1–3 ms target was an aspiration copied before any measurement.
Tightening suggestion for the next CLAUDE.md update: the ceiling
column should be `≤ 1 ms` for cache-hit on a warm isolate; anything
above that is a regression (either the Cache API slowed, our JSON
round-trip got bloated, or the pipeline's early-return for cache hit
regressed).

## Repro

```sh
cd examples/minimal-worker
node scripts/gen-bench-fixture.mjs   # refresh the inlined fixture
pnpm exec wrangler dev --port 8788 --local

# In another terminal, warm the isolate first with any request, then:
curl 'http://localhost:8788/bench?iters=5000&path=cache'
curl 'http://localhost:8788/bench?iters=500&path=dc'
curl 'http://localhost:8788/bench?iters=500&path=full'
```
